Installation

Concept Core is distributed as the Composer package php-concept/core. You can start in two ways: with the ready-made Concept Skeleton (recommended for new projects) or by adding core only to your own application.

Requirements

For php-concept/core you need:

  • PHP 8.4 or newer
  • Composer 2.x
  • PHP extensions: json, mbstring, dom, xml
  • pdo — if you use a database (Eloquent, migrations)

Concept Skeleton additionally expects pdo_mysql, mysqli, zip (see Dockerfile in the skeleton repository). For local development without Docker, PHP with these extensions and a web server with document root on public/ is enough.

Concept Skeleton

Concept Skeleton is a minimal starter application: core already wired, bootstrap, providers, / route with a landing page, Twig, config, and a CLI example. The fastest way to see Concept Core in action.

Step 1. Clone and dependencies

git clone https://github.com/php-concept/concept-skeleton.git my-app
cd my-app

composer install

Step 2. Environment

cp .env.example .env

Edit .env: APP_ENV, APP_DEBUG, DB settings, etc. More details — in Configuration.

Step 3. Web server

Document root must point to the public/ directory. Front controller — public/index.php; it loads autoload and bootstrap/app.php.

For a quick local check:

php -S localhost:8080 -t public

Open http://localhost:8080/ — the Concept Skeleton landing page should appear.

Docker (optional)

cp docker-compose.yml.example docker-compose.yml
# Edit container_name and volumes
docker compose up -d

docker exec -it container_name bash
composer install

Apache vhost in skeleton: docker/apache/vhost.conf (DocumentRoot → public/).

What is already configured in skeleton

  • php-concept/core in composer.json
  • bootstrap/paths.php, bootstrap/app.php, HTTP and CLI providers
  • Routes in routes/web.php, views in resources/views/frontend/
  • PSR-4: namespace Concept\ → src/
  • CLI: php bin/console.php route:list

Bootstrap, entry points, and paths.php are described in detail in Setup.

Core only

Install only php-concept/core if you integrate Concept into an existing project or assemble the directory structure yourself. Core does not create files for you — after Composer you must manually set up bootstrap, config, routes, and entry points. 

composer require php-concept/core

Then create at minimum:

  • bootstrap/paths.php, bootstrap/app.php, bootstrap/providers/app.php
  • public/index.php (HTTP) and optionally bin/console.php (CLI)
  • config/, routes/, storage/ — depending on enabled providers

Step-by-step bootstrap, paths.php, providers, and console — in Setup. Directory organization options — in Directory structure.

Skeleton vs core only

  • Skeleton — ready project; clone, configure .env, start working.
  • Core only — full control over structure; more manual bootstrap and config work.
