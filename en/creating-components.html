Creating a Component

Step-by-step walkthrough based on AuthAdmin in the skeleton: admin login, dashboard, user CRUD, migrations, seeder, and CLI command.

1. Component class

class AuthAdminComponent extends BaseComponent implements ComponentInterface
{
    protected string $componentDir = __DIR__;
}

2. config.php

return [
    'name' => 'AuthAdmin',
    'migrations' => ['paths' => ['src/Components/AuthAdmin/Database/Migrations']],
    'view' => [
        'paths' => ['auth-admin' => '/src/Components/AuthAdmin/Views'],
        'contexts' => ['/admin' => 'dashboard'],
    ],
    'assets' => [
        'src/Components/AuthAdmin/Assets/admin-tokens.js'
            => 'public/components/auth-admin/js/admin-tokens.js',
    ],
];

3. Register in config/components.php

AuthAdminComponent::class => AuthAdminComponent::class,

4. routes.php

Public /admin/login without auth middleware; protected groups use AuthMiddleware::class.

5–7. Middleware, controllers, views

AdminController uses LoginRequest, AuthService, and @auth-admin/login extending @dashboard/layouts/auth.twig.

8. Database

php bin/console.php db:migrate
php bin/console.php db:seed -c "Concept\Components\AuthAdmin\Database\Seeders\UserSeeder"

For a single seeder, pass the full FQCN via -c (quoted). Short names like UserSeeder do not work.

9. Assets & CLI

php bin/console.php component:publish-assets
php bin/console.php component:list

See CLI & Assets for details.

Checklist

  1. Component class + config.php
  2. Entry in config/components.php
  3. routes.php, controllers, services, views
  4. Migrations / seeders / commands / assets as needed
  5. Run migrate, publish assets, route:list
