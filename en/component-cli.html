CLI & Assets

component:list

php bin/console.php component:list

Lists active modules (name, version, class, description).

component:publish-assets

Copies paths from each component’s assets map in config.php into public/.

php bin/console.php component:publish-assets

Module commands

Declare command classes in config.php → 'commands' => [UserListCommand::class]. Registered automatically on boot.

Migrations & seeders

php bin/console.php db:migrate
php bin/console.php db:seed
php bin/console.php db:seed -c "Concept\Components\AuthAdmin\Database\Seeders\UserSeeder"

db:seed without options runs all registered seeders. For one seeder, use -c / --class with the full FQCN (quoted). Short class names are not supported.

Component migration paths are merged with global paths from config/migrations.php.

Per-environment components

Use config/dev/components.php to add DebugBar only in development.

Events

ComponentRegistering is dispatched for each module during boot — useful for telemetry or validation hooks.
