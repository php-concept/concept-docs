CLI & Assets

Concept Core provides console commands to list registered modules and publish static files to public/. Component commands are wired automatically during ComponentsServiceProvider::boot().

component:list

php bin/console.php component:list

Prints a table: Name, Version, Class, Description — from each active module’s config.php.

If a component is missing, check config/components.php and that ComponentsServiceProvider is registered last among HTTP providers.

component:publish-assets

Copies files and directories from the assets map in the component’s config.php:

// src/Components/AuthAdmin/config.php
'assets' => [
    'src/Components/AuthAdmin/Assets/admin-tokens.js'
        => 'public/components/auth-admin/js/admin-tokens.js',
],
php bin/console.php component:publish-assets
  • Paths are relative to the project root (PathManager::root())
  • Files — copy; directories — mirror with overwrite
  • All active components are processed in one run

After changing JS/CSS under Component/Assets/, run the command again before deploy.

Module commands

Declare classes in config.php:

// AuthAdmin
'commands' => [
    UserListCommand::class,
],

The command is a regular Symfony Console class registered in the container (autowiring). Example:

// Commands/UserListCommand.php
final class UserListCommand extends Command
{
    protected function configure(): void
    {
        $this->setName('auth:user-list');
    }
    // execute() ...
}

See Creating Commands.

Migrations and seeders

Paths from migrations.paths and classes from seeders.list are added to core registries:

php bin/console.php db:migration:list
php bin/console.php db:migrate
php bin/console.php db:seed
php bin/console.php db:seed -c "Concept\Components\AuthAdmin\Database\Seeders\UserSeeder"

db:seed without options runs all registered seeders. For one seeder use full FQCN in -c / --class (quoted). Short class names do not work.

Global migrations from config/migrations.php and component migrations appear in one list.

Different component sets per environment

// config/dev/components.php — only when APP_ENV=dev
return [
    'components' => [
        DebugBarComponent::class => DebugBarComponent::class,
    ],
];

ConfigServiceProvider merges overrides from config/{APP_ENV}/ — in dev, DebugBar is added without changing the production list.

Events

During boot each module dispatches ComponentRegistering — subscribe for telemetry or checks:

$dispatcher->addListener(
    EventName::FRAMEWORK_COMPONENT_REGISTERING,
    function (ComponentRegistering $event) {
        $component = $event->component;
        // $component->name(), routes(), ...
    }
);
