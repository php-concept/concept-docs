Configuration

Before this section, read Setup — bootstrap and paths.php. Concept Core loads settings from PHP files in config/, then applies environment overrides (APP_ENV) and variables from .env. Access in code is through ConfigInterface (DI container).

Loading order

  1. All PHP files from config/ (Noodlehaus Config library).
  2. The .env file in the project root (vlucas/phpdotenv).
  3. Override from config/{APP_ENV}/ if that folder exists (e.g. config/dev/).
  4. Merge .env variables into config (highest priority).

After loading, the timezone is set from app.timezone.

The APP_ENV value is read from .env before merging variables into config and is used only to select the override folder config/{APP_ENV}/.

.env → config mapping

Each environment variable is converted to a config key using the “first underscore” rule: the name is split into two parts (explode('_', ..., 2) limit), lowercased, then joined with a dot. 

APP_DEBUG=true     → app.debug
DB_HOST=127.0.0.1   → db.host
LOG_MAX_FILES=14    → log.max_files

Deeper nesting via .env is not supported: APP_FOO_BAR_BAZ becomes app.foo_bar_baz, not a nested array. Complex structures belong in config/ PHP files.

Boolean values in .env (true, false, 0, 1) are read correctly through getBool().

Override by APP_ENV

If the directory config/{APP_ENV value}/ exists, all PHP files from it are merged on top of the base config/. File names and array structure are the same — only the needed keys change.

By default:

  • config/dev/ — app.debug = true
  • config/production/ — app.debug = false
config/
├── app.php
├── dev/
│   ├── app.php          # debug: true
│   └── components.php   # component list
└── production/
    ├── app.php          # debug: false
    └── components.php   # empty list

Access in code

Inject Concept\Core\Components\Config\Contracts\ConfigInterface:

public function __construct(
    private readonly ConfigInterface $config,
) {}

$debug = $this->config->getBool('app.debug');
$name = $this->config->getString('app.name');
$routes = $this->config->get('routes', []);

Methods: get(), has(), set(), all(), getString(), getInt(), getBool().

Development tips

  • Locally set APP_ENV=dev and APP_DEBUG=true in .env.
  • Secrets (passwords, API keys) — only in .env, not in config/*.php.
  • Shared defaults — in config/; environment differences — in config/dev/ or config/production/.
  • New env variables follow the convention GROUP_NAME → group.name.
  • Clear Twig cache after template changes: php bin/console.php view:clear.
