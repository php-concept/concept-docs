Contributing

Concept Core, Concept Skeleton, and this documentation are separate repositories in the php-concept ecosystem. Below is how to contribute changes to code and docs consistently.

Repositories

ProjectPurpose
concept-core Core: HTTP, DI, validation, DB, views, CLI
concept-skeleton Starter application, config, bootstrap
concept-docs Static HTML documentation (this site)

Documentation (concept-docs)

  • Two languages: uk/ and en/; primary content is often written in Ukrainian first.
  • Before updating a page, verify against code in code/core and code/skeleton (mirrors for doc authors).
  • Do not change code/ content in documentation PRs — only HTML/CSS/JS in the repository root.
  • Navigation: assets/js/nav-uk.js, nav-en.js; new page — slug in sections and nextPrev.
  • Detailed rules for AI agents — AGENTS.md in the repository root.

Core and skeleton code

  • PHP 8.4+, declare(strict_types=1) in public API.
  • Core changes — via PR to php-concept/core; skeleton pulls new version through Composer.
  • After API changes, update corresponding doc sections (routing, providers, config).
  • Tests and CI — per each repository setup (check PR status on GitHub).

Documentation style

  • Examples from real skeleton code, not invented APIs.
  • Do not duplicate TOC in text — “On this page” is already on the right.
  • Code in <pre><code class="language-php"> for Highlight.js.

Questions and suggestions — via Issues/Discussions on the relevant GitHub repository.
