CSRF Protection

Concept Core protects state-changing HTTP requests from cross-site request forgery using a synchronizer token stored in the session. Verification runs in VerifyCsrfTokenMiddleware before the controller; CsrfTokenManager issues tokens for forms and AJAX.

See also: Middleware → CSRF, Views (csrf_token in Twig).

How it works

  1. On first use, CsrfTokenManager::getToken() generates a 64-character hex token (random_bytes(32)) and stores it in the session under SessionKey::CSRF_TOKEN (_csrf_token).
  2. ShareViewDataMiddleware exposes the token in Twig as {{ csrf_token }}.
  3. For POST, PUT, and DELETE, middleware compares the request token with the session value using hash_equals().
  4. If the token is missing or invalid, CsrfException is thrown (HTTP 419 Page Expired).

Enabling middleware

In routes/web.php (or on a route group), register verification after the validation handler but before state changes:

$router->lazyMiddleware(HandleValidationExceptionMiddleware::class);
$router->lazyMiddleware(VerifyCsrfTokenMiddleware::class);
$router->lazyMiddleware(StorePreviousUrlMiddleware::class);
$router->lazyMiddleware(ShareViewDataMiddleware::class);

ShareViewDataMiddleware must be in the stack so templates receive a current token. Sessions are provided by SessionServiceProvider in bootstrap/providers/app.php.

HTML forms

Every form that mutates data should include a hidden field named _csrf_token:

<form method="post" action="{{ url('cabinet.profile.update') }}">
    <input type="hidden" name="_csrf_token" value="{{ csrf_token }}">
    {# ... form fields ... #}
</form>

The _csrf_token field is excluded from validation data in FormRequest (global $globalExcept), so it never lands in your DTO.

AJAX and browser APIs

Middleware also accepts the token from headers (after the parsed body):

  • X-CSRF-TOKEN — raw value
  • X-XSRF-TOKEN — passed through urldecode()
fetch('/api/profile', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/json',
        'X-CSRF-TOKEN': document.querySelector('meta[name="csrf-token"]').content
    },
    body: JSON.stringify(payload)
});

For stateless external APIs, omit CSRF on a separate route group and use Bearer tokens, API keys, or similar.

Errors and responses

CsrfException uses HttpStatusCode::PAGE_EXPIRED (419). The error handler returns HTML or JSON depending on the request — see Responses.

Common causes of 419:

  • Expired or missing session cookie
  • Stale form page after the session token rotated
  • Cached page submit without a fresh csrf_token

Programmatic access

use Concept\Core\Components\Csrf\CsrfTokenManager;

/** @var CsrfTokenManager $csrf */
$csrf = $container->get(CsrfTokenManager::class);
$token = $csrf->getToken();
$valid = $csrf->validate($submittedToken);

GET, HEAD, and OPTIONS are not checked. Never perform state changes via GET.
