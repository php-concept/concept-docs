Component Concept

A component in Concept Core is an isolated application feature module (modular monolith): its own routes, templates, migrations, CLI commands, and optional DI providers. The core assembles active components via ComponentRegistry and ComponentsServiceProvider.

See also: directory structure, component providers, creating a component (AuthAdmin).

Why components

  • Package by feature — admin, cabinet, and catalog code live together.
  • Enable/disable — one line in config/components.php.
  • Team boundaries — changes under AuthAdmin/ rarely touch UserCabinet/.
  • Shared core — HTTP, validation, Twig, and DB stay in Concept Core and App/.

Where to put code

src/
├── App/                    # shared application code
└── Components/
    └── AuthAdmin/          # example module
        ├── AuthAdminComponent.php
        ├── config.php
        ├── routes.php
        ├── Controllers/  Views/  Database/  ...

config/components.php       # active modules
resources/views/            # shared layouts (@dashboard, @frontend)
routes/web.php              # global routes + middleware stack

ComponentInterface

Component classes implement ComponentInterface (via BaseComponent in the skeleton). Key methods: routes(), viewPaths(), viewContexts(), commands(), migrations(), seeders(), assets(), providers().

Metadata and lists are declared in the module’s config.php.

Registration

// config/components.php
return [
    'components' => [
        AuthAdminComponent::class => AuthAdminComponent::class,
    ],
];

Dev-only modules can be added in config/dev/components.php (e.g. DebugBar).

Boot sequence

ComponentsServiceProvider::boot() registers commands, migrations, seeders, nested providers, requires routes.php files, and (in HTTP) Twig paths/extensions/contexts.

Moving existing code into a component

  1. Create src/Components/MyFeature/ following AuthAdmin layout.
  2. Move controllers, models, requests, views; update namespaces.
  3. Extract routes to MyFeature/routes.php.
  4. Add MyFeatureComponent + config.php.
  5. Register in config/components.php.
  6. Keep shared code in App/ and resources/views/.
