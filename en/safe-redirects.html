Safe Redirects

Open redirects (?next=https://evil.com) are a common vulnerability. Concept Core limits “go back” and fallback targets to in-app URLs via StorePreviousUrlMiddleware and ResponseFactory::back().

StorePreviousUrlMiddleware

The middleware stores two URLs in the session:

  • SessionKey::CURRENT_URL (_url_current) — latest GET page (not AJAX)
  • SessionKey::PREVIOUS_URL (_url_previous) — previous GET page

Each request gets a safe “back” URL on the PSR-7 attribute:

  • GET — PREVIOUS_URL (page you came from)
  • POST (and other methods) — CURRENT_URL (form page)
// RequestAttribute::SAFE_BACK_URL on the request
$router->lazyMiddleware(StorePreviousUrlMiddleware::class);

AJAX requests (X-Requested-With: XMLHttpRequest) do not update the URL chain, so background polls do not break “back” after a form submit.

ResponseFactory::back()

Controllers or middleware often redirect back after validation:

return $responseFactory->back();

Target resolution order:

  1. Referer header, if internal
  2. SAFE_BACK_URL request attribute
  3. Fallback — default / (second argument to back())
return $responseFactory->back(
    status: 302,
    fallback: '/cabinet'
);

What counts as internal

isInternalUrl() on ResponseFactory:

  • Relative paths starting with / are allowed
  • Absolute URLs are allowed only when host matches the current request host
  • External domains, javascript:, and other-site protocols are rejected → fallback

This blocks open redirects even if an attacker forges Referer or session data (middleware only writes real application URIs into the session).

After validation errors

HandleValidationExceptionMiddleware flashes errors and calls back() for HTML — the user returns to the form URL without redirect($_GET['return']).

See Requests & Validation and Middleware.

Named routes

For fixed targets, named redirects are safer — the host is controlled by the URL generator:

return $responseFactory->redirectByName('cabinet.dashboard');

Use back() for “return to the form” UX; use redirectByName() when the destination is always known (login, dashboard).

Never redirect to raw user input. Whitelist paths or route names if you need a ?next= parameter.
