Safe Redirects

Open redirects (?next=https://evil.com) are a common vulnerability. Concept Core limits “go back” and fallback targets to in-app URLs via StorePreviousUrlMiddleware and ResponseFactory::back().

StorePreviousUrlMiddleware

The middleware tracks two session keys:

  • SessionKey::CURRENT_URL — latest non-AJAX GET page
  • SessionKey::PREVIOUS_URL — page before that

Each request gets RequestAttribute::SAFE_BACK_URL:

  • GET → previous page URL
  • POST (and others) → current form page URL
$router->lazyMiddleware(StorePreviousUrlMiddleware::class);

AJAX requests (X-Requested-With: XMLHttpRequest) do not update the URL chain.

ResponseFactory::back()

return $responseFactory->back(status: 302, fallback: '/cabinet');

Target resolution order:

  1. Referer header, if internal
  2. SAFE_BACK_URL request attribute
  3. Fallback path (default /)

Internal URL rules

  • Relative paths starting with / are allowed
  • Absolute URLs must match the current request host
  • External hosts are rejected → fallback

After validation errors

HandleValidationExceptionMiddleware flashes errors and calls back() for HTML — users return to the form without trusting a user-supplied return URL.

Named routes

return $responseFactory->redirectByName('cabinet.dashboard');

Use back() for form UX; use redirectByName() when the destination is fixed.

Never redirect to raw user input. Whitelist paths or route names if you need a ?next= parameter.
