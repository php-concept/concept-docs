Upgrade guide

Recommended path to upgrade a Concept Skeleton-based application after a new php-concept/core release. Always check release notes for the specific version.

1. Composer

composer update php-concept/core --with-dependencies

Pin a range in composer.json (e.g. ^1.0). After updating, review the diff in vendor/php-concept/core or the changelog on GitHub.

2. Service providers

Compare bootstrap/providers/app.php and console.php with the current core list. New providers (e.g. TelemetryServiceProvider, LocaleServiceProvider) must be added in the correct order.

3. Configuration

  • Check new files in skeleton config/ (telemetry, masking, locale, etc.).
  • Keys from Concept\Core\Foundation\ConfigKey — in Configuration.
  • Environment override: config/{APP_ENV}/.
  • Update .env.example if new variables appeared.

4. Typical API changes

  • URL — UrlGeneratorInterface::uri(), url(), base() (not Laravel path()/route()).
  • Masking — DataMaskerInterface, DataMaskerServiceProvider.
  • Telemetry — TelemetryCollector, not EventServiceProvider — see Telemetry.
  • Session — session.cookie_domain, session.cookie_samesite (not session.domain).

5. Post-upgrade verification

  1. php bin/console.php route:list — routes in place
  2. php bin/console.php db:migrate — migrations (if changed)
  3. Home page, login, forms with CSRF, redirect after POST
  4. In dev: APP_DEBUG=true, optionally telemetry.enabled=true

If after upgrade a service is “not found” in the container — check provider order and component presence in config/components.php (service container).
