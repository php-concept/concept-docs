Release notes

Brief overview of the Concept Core 1.x line and related documentation. Detailed package release changelog — on GitHub Releases.

Concept Core 1.x

First stable engine line focused on PSR and modularity:

  • PSR-7 HTTP, PSR-11 DI (League Container + Reflection), PSR-15 middleware
  • League Route, RouteStrategy (FormRequest, route params, interceptors)
  • Illuminate Database (Eloquent, migrations, pagination)
  • Rakit validation, Valinor DTO casting, Twig/Plates views
  • Component registration: routes, migrationPaths, seeders, views, CLI
  • CSRF, data masker, safe redirects, telemetry hooks
  • Symfony Console, Whoops / production fallbacks
  • Strict types PHP 8.4+

Concept Skeleton

Minimal starter application: bootstrap/, config/, routes/web.php, one controller and frontend template.

Documentation 1.x

Static site (Bootstrap 5, Highlight.js): sections from installation through components, Ukrainian and English versions. Content is verified against code in code/core and code/skeleton.

Reference components

The documentation examples are based on reference components shipped with skeleton-demo (code/skeleton-demo/src/Components/). Each implements ComponentInterface directly:

ComponentPurposeKey methods
AuthAdmin Admin authentication, dashboard, user CRUD routes(), viewPaths(), commands(), assets()
Acl Route-level ACL, admin matrix UI, AclRouteAuthorization providers(), seeders(), migrationPaths()
SettingsManager Centralized application settings storage providers(), viewPaths(), assets()
DebugBar PHP Debug Bar integration (dev only) providers(), assets()

See Creating a component for a step-by-step walkthrough using AuthAdmin, and Route interceptors for the Acl component.

When updating php-concept/core in a project, also see Upgrade guide.
