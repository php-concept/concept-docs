Створення консольних команд

Команда — клас, що успадковує Symfony\Component\Console\Command\Command. Залежності передаються через конструктор (автовайринг контейнера); логіка — у методі execute().

Документація Symfony: Creating a Command, Input, Output styling.

Мінімальна команда

<?php declare(strict_types=1);

namespace Concept\App\Console\Commands;

use Symfony\Component\Console\Command\Command;
use Symfony\Component\Console\Input\InputInterface;
use Symfony\Component\Console\Output\OutputInterface;
use Symfony\Component\Console\Style\SymfonyStyle;

final class GreetCommand extends Command
{
    protected function configure(): void
    {
        $this->setName('app:greet')
            ->setDescription('Print a greeting');
    }

    protected function execute(InputInterface $input, OutputInterface $output): int
    {
        $io = new SymfonyStyle($input, $output);
        $io->success('Hello from Concept Core');

        return Command::SUCCESS;
    }
}

Реєстрація в застосунку

Додайте клас у config/commands.php:

return [
    'commands' => [
        // ...команди ядра
        GreetCommand::class,
    ],
];

ConsoleServiceProvider робить $container->get(GreetCommand::class) і addCommand(). Якщо в конструкторі потрібні сервіси — вони резолвляться автоматично (автовайринг). 

php bin/console.php app:greet

Аргументи та опції

Приклад з опцією (як user:list):

use Symfony\Component\Console\Input\InputOption;

protected function configure(): void
{
    $this->setName('user:list')
        ->setDescription('Display registered users')
        ->addOption('limit', 'l', InputOption::VALUE_OPTIONAL, 'Max rows', 10);
}

protected function execute(InputInterface $input, OutputInterface $output): int
{
    $io = new SymfonyStyle($input, $output);
    $limit = (int) $input->getOption('limit');

    $users = $this->userModel->newQuery()->limit($limit)->get();
    $io->table(['id', 'email', 'name'], /* ... */);

    return Command::SUCCESS;
}

Залежності в конструкторі:

public function __construct(
    private readonly UserModel $userModel,
) {
    parent::__construct();
}

Приклад з ядра: DI в команді

RouteListCommand отримує Router з контейнера:

public function __construct(private readonly Router $router)
{
    parent::__construct();
}

DbMigrateCommand — Migrator + MigrationRegistry. Складні команди ядра лежать у Concept\Core\Console\Commands.

Команда в компоненті

  1. Створіть Components/MyModule/Commands/MyCommand.php
  2. Додайте клас у commands у Component/config.php
  3. Переконайтеся, що компонент увімкнено в config/components.php
// src/Components/AuthAdmin/config.php
'commands' => [
    UserListCommand::class,
],

Реєстрація відбувається в ComponentsServiceProvider::registerConsoleCommands() під час boot() — після того, як ConsoleApplication уже створений провайдером консолі.

Вивід і помилки

$io = new SymfonyStyle($input, $output);

$io->title('Import started');
$io->writeln('Processing...');
$io->success('Done');
$io->warning('Partial import');
$io->error('Failed');

// таблиця
$io->table(['Column A', 'Column B'], $rows);

// progress (довгі цикли)
$io->progressStart(count($items));
foreach ($items as $item) {
    // ...
    $io->progressAdvance();
}
$io->progressFinish();

При винятку повертайте Command::FAILURE або дозвольте Symfony вивести stack trace (у dev з Whoops для CLI — plain text).

Чеклист

  1. Унікальне ім’я команди (module:action, наприклад billing:sync)
  2. configure() — name, description, arguments/options
  3. execute() — коротка orchestration, важке в сервісі
  4. Реєстрація в config/commands.php або Component/config.php
  5. Перевірка: php bin/console.php list і help your:command

Не дублюйте HTTP-контролери в CLI: винесіть спільну логіку в Services/ і викличте її і з web, і з command.
