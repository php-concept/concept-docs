Меню

Встановлення

Concept Core постачається як Composer-пакет php-concept/core. Почати можна двома шляхами: через готовий Concept Skeleton (рекомендовано для нових проєктів) або підключивши ядро окремо до власного застосунку.

Вимоги

Для php-concept/core потрібні:

  • PHP 8.4 або новіший
  • Composer 2.x
  • Розширення PHP: json, mbstring, dom, xml
  • pdo — якщо використовуєте базу даних (Eloquent, migrations)

Concept Skeleton додатково очікує pdo_mysql, mysqli, zip (див. Dockerfile у репозиторії skeleton). Для локальної розробки без Docker достатньо PHP з цими розширеннями та веб-сервера з document root на public/.

Concept Skeleton

Concept Skeleton — мінімальний стартовий застосунок: уже підключене ядро, bootstrap, провайдери, маршрут / з landing page, Twig, конфіг і приклад CLI. Найшвидший спосіб побачити Concept Core в роботі.

Крок 1. Клонування та залежності

git clone https://github.com/php-concept/concept-skeleton.git my-app
cd my-app

composer install

Крок 2. Середовище

cp .env.example .env

Відредагуйте .env: APP_ENV, APP_DEBUG, параметри БД тощо. Детальніше — у розділі Конфігурація.

Крок 3. Веб-сервер

Document root має вказувати на каталог public/. Front controller — public/index.php; він підключає autoload і bootstrap/app.php.

Для швидкої перевірки локально:

php -S localhost:8080 -t public

Відкрийте http://localhost:8080/ — має з’явитися landing page Concept Skeleton.

Docker (опційно)

cp docker-compose.yml.example docker-compose.yml
# Відредагуйте container_name та volumes
docker compose up -d

docker exec -it container_name bash
composer install

Apache vhost у skeleton: docker/apache/vhost.conf (DocumentRoot → public/).

Що вже налаштовано в skeleton

  • php-concept/core у composer.json
  • bootstrap/paths.php, bootstrap/app.php, провайдери HTTP і CLI
  • Маршрути в routes/web.php, views у resources/views/frontend/
  • PSR-4: namespace Concept\ → src/
  • CLI: php bin/console.php route:list

Bootstrap, точки входу та paths.php детально описані в Налаштування.

Ядро окремо

Підключайте лише php-concept/core, якщо інтегруєте Concept у наявний проєкт або збираєте структуру каталогів самостійно. Ядро не створює файли за вас — після Composer потрібно вручну зібрати bootstrap, config, routes і точки входу. 

composer require php-concept/core

Далі створіть мінімум:

  • bootstrap/paths.php, bootstrap/app.php, bootstrap/providers/app.php
  • public/index.php (HTTP) і за потреби bin/console.php (CLI)
  • config/, routes/, storage/ — залежно від увімкнених провайдерів

Покроковий розбір bootstrap, paths.php, провайдерів і консолі — у розділі Налаштування. Варіанти організації каталогів — у Структура каталогів.

Skeleton vs ядро окремо

  • Skeleton — готовий проєкт; клонуєте, налаштовуєте .env, працюєте.
  • Ядро окремо — повний контроль над структурою; більше ручної роботи з bootstrap і config.

Наступні кроки

  • Skeleton: Конфігурація → Життєвий цикл запиту
  • Ядро окремо: Налаштування → Конфігурація
  • Структура каталогів — організація src/
