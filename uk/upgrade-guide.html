Оновлення

Рекомендований шлях оновлення застосунку на базі Concept Skeleton після виходу нової версії php-concept/core. Завжди перевіряйте release notes конкретної версії.

1. Composer

composer update php-concept/core --with-dependencies

Зафіксуйте діапазон у composer.json (наприклад ^1.0). Після оновлення перегляньте diff у vendor/php-concept/core або changelog на GitHub.

2. Service providers

Порівняйте bootstrap/providers/app.php і console.php із актуальним списком ядра. Нові провайдери (наприклад TelemetryServiceProvider, LocaleServiceProvider) потрібно додати в правильному порядку.

3. Конфігурація

  • Перевірте нові файли в config/ skeleton (telemetry, masking, locale тощо).
  • Ключі з Concept\Core\Foundation\ConfigKey — у Конфігурація.
  • Override для середовища: config/{APP_ENV}/.
  • Оновіть .env.example, якщо з’явилися нові змінні.

4. Відповідність поточному API ядра (v1.1+)

  • Конфігурація — ключі в Конфігурація та Concept\Core\Foundation\ConfigKey.
  • Простори імен — Components перейменовано на Services, RequestAttribute перенесено в Http\Requests.
  • Інтерфейс компонентів — перейменуйте migrations() на migrationPaths(); видаліть Component/config.php і BaseComponent — реалізуйте ComponentInterface напряму з методами migrationPaths(), viewPaths(), seeders(), commands(), assets(), providers() (див. Створення компонентів).
  • Flash Bag — FlashBagInterface тепер має типізовані методи: addError(), addSuccess() тощо.
  • Телеметрія — TelemetryCollector, події та прапорці в Телеметрія.
  • URL — UrlGeneratorInterface::uri(), url(), base() (не Laravel path()/route()).
  • Маскування — DataMaskerInterface, DataMaskerServiceProvider.
  • Session — session.cookie_domain, session.cookie_samesite (не session.domain).

5. Перевірка після оновлення

  1. php bin/console.php route:list — маршрути на місці
  2. php bin/console.php db:migrate — міграції (якщо змінювались)
  3. Головна сторінка, login, форми з CSRF, redirect після POST
  4. У dev: APP_DEBUG=true, за потреби telemetry.enabled=true

Якщо після оновлення сервіс «не знаходиться» в контейнері — перевірте порядок провайдерів і наявність компонента в config/components.php (контейнер сервісів).
