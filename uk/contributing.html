Участь у розробці

Concept Core, Concept Skeleton і ця документація — окремі репозиторії екосистеми php-concept. Нижче — як узгоджено вносити зміни в код і в docs.

Репозиторії

ПроєктПризначення
concept-core Ядро: HTTP, DI, валідація, БД, views, CLI
concept-skeleton Стартовий застосунок, конфіг, bootstrap
concept-docs Статична HTML-документація (цей сайт)

Документація (concept-docs)

  • Дві мови: uk/ і en/; основний зміст часто спочатку в українській версії.
  • Перед оновленням сторінки звіряйтеся з кодом у code/core і code/skeleton (дзеркала для авторів docs).
  • Не змінюйте вміст code/ у PR документації — лише HTML/CSS/JS у корені репозиторію.
  • Навігація: assets/js/nav-uk.js, nav-en.js; нова сторінка — slug у секціях і nextPrev.
  • Детальні правила для ШІ-агентів — AGENTS.md у корені репозиторію.

Код ядра та skeleton

  • PHP 8.4+, declare(strict_types=1) у публічному API.
  • Зміни в core — через PR у php-concept/core; skeleton підтягує нову версію через Composer.
  • Після змін API оновіть відповідні розділи docs (маршрутизація, провайдери, конфіг).
  • Тести та CI — за налаштуванням кожного репозиторію (перевіряйте статус PR на GitHub).

Стиль документації

  • Приклади з реального коду skeleton, не вигадані API.
  • Не дублюйте TOC у тексті — праворуч уже є блок «На цій сторінці».
  • Код у <pre><code class="language-php"> для Highlight.js.

Питання та пропозиції — через Issues/Discussions відповідного репозиторію на GitHub.
