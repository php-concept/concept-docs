Робота з консоллю

CLI застосунку побудований на Symfony Console. Точка входу — bin/console.php; команди реєструються через DI-контейнер і ConsoleServiceProvider.

Див. також: налаштування → bin/console.php, провайдери для CLI, створення власних команд.

Запуск

З кореня проєкту:

php bin/console.php

# або через Composer (script у composer.json)
composer console

# довідка
php bin/console.php list

# довідка по одній команді
php bin/console.php help db:migrate

Bootstrap CLI відрізняється від HTTP: підключається bootstrap/providers/console.php (без View/Validation для веб-стеку, з ConsoleServiceProvider). Детальніше — Setup.

Як це влаштовано

// bin/console.php
$app = App::create($rootPath, $paths);
$app->registerServiceProviders([$rootPath . '/bootstrap/providers/console.php']);

$console = $app->getContainer()->get(ConsoleApplication::class);
$console->run();
  1. ConsoleServiceProvider створює Symfony\Component\Console\Application
  2. Команди з config/commands.php резолвяться з контейнера і додаються в application
  3. ComponentsServiceProvider::boot() додає команди з config.php кожного компонента

Команди ядра

Реєстрація в config/commands.php:

КомандаПризначення
db:migrateВиконати міграції
db:migration:listСписок міграцій
db:rollbackВідкат останнього batch міграцій
db:seedУсі сидери; -c "Fully\Qualified\Class" — один (повний FQCN)
db:seeders:listСписок зареєстрованих сидерів
route:listТаблиця маршрутів (-F — повні імена middleware)
view:clearОчистити кеш Twig
component:listСписок увімкнених компонентів
component:publish-assetsПублікація assets компонентів у public/

Приклади

php bin/console.php db:migrate
php bin/console.php db:seed
php bin/console.php db:seed -c "Concept\Components\AuthAdmin\Database\Seeders\UserSeeder"
php bin/console.php route:list
php bin/console.php route:list -F
php bin/console.php component:list
php bin/console.php view:clear

Команди компонентів

Компонент додає команди в секції commands свого config.php. Під час ComponentsServiceProvider::boot() вони реєструються в тому ж ConsoleApplication.

Коди завершення

Команди повертають Command::SUCCESS (0) або Command::FAILURE (1). У скриптах CI перевіряйте exit code після php bin/console.php ....

Оточення та конфіг

CLI використовує той самий .env і ConfigServiceProvider, що й HTTP. APP_ENV визначає оверрайди config/{env}/ і список компонентів (наприклад DebugBar лише в config/dev/components.php).

Для міграцій і сидерів потрібне коректне підключення БД у config/db.php (або через .env).

Консоль — для обслуговування (міграції, кеш, діагностика) і локальної розробки. Довготривалу бізнес-логіку краше виносити в сервіси, а в command лишати тонкий виклик.
