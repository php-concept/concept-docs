Маршрутизація

Concept Core використовує League Route поверх PSR-7. Маршрути описуються у PHP-файлах; один спільний екземпляр Router обслуговує весь застосунок.

Контекст: життєвий цикл запиту, config/routes.php.

Підключення файлів маршрутів

У config/routes.php перелічіть шляхи до файлів, які підключає HttpServiceProvider:

return [
    'routes' => [
        dirname(__DIR__) . '/routes/web.php',
    ],
];

Додаткові файли з компонентів підключаються під час ComponentsServiceProvider::boot() — кожен компонент може повернути шлях до власного routes.php (див. структура з компонентами).

У кожному файлі маршрутів доступна змінна $router (тип League\Route\Router):

<?php declare(strict_types=1);

use League\Route\Router;

/** @var Router $router */

$router->get('/path', $handler);

Обробники (handlers)

League Route підтримує кілька форм callable:

Контролер [Class, method]

Рекомендований варіант — метод контролера, який резолвиться з DI-контейнера:

use Concept\App\Controllers\IndexController;

$router->get('/', [IndexController::class, 'index'])->setName('home');

Аргументи методу підставляє RouteStrategy (FormRequest, ServerRequestInterface, параметри URL, сервіси з контейнера). Детальніше — Контролери.

Closure

$router->get('/health', fn () => new Response(/* ... */));

Для простих endpoint-ів; для складної логіки краще винести в контролер.

HTTP-методи

Метод RouterHTTP
get()GET
post()POST
put()PUT
patch()PATCH
delete()DELETE
map()кілька методів

Параметри URL

Сегменти в фігурних дужках потрапляють у атрибути запиту та в аргументи контролера:

$router->get('/admin/users/edit/{id:number}', [UserController::class, 'edit']);

Суфікс :number — обмеження League Route (лише цифри). Інші: :alpha, :alphanum, :segment тощо. Без суфікса — довільний сегмент ({token}).

У контролері параметр з тією ж назвою та типом (через Caster):

public function edit(int $id): ResponseInterface

Групи маршрутів

Спільний префікс URI та middleware для групи:

use League\Route\RouteGroup;
use Concept\Components\AuthAdmin\Middlewares\AuthMiddleware;

$router->group('/admin', function (RouteGroup $route) {
    $route->get('/', [AdminController::class, 'index'])->setName('admin.home');
    $route->get('/logout', [AdminController::class, 'logout'])->setName('admin.logout');
})->lazyMiddleware(AuthMiddleware::class);

URL /admin та /admin/logout проходять через AuthMiddleware. Вкладені групи підтримуються.

Middleware на маршрутах

Глобальний — для всіх маршрутів у файлі (зазвичай на початку routes/web.php):

$router->lazyMiddleware(VerifyCsrfTokenMiddleware::class);
$router->lazyMiddleware(ShareViewDataMiddleware::class);

На групу або маршрут — ланцюжок доповнюється для збігів у цій області:

$router->post('/projects/store', [ProjectController::class, 'store'])
    ->setName('projects.store')
    ->lazyMiddleware(ActiveUserMiddleware::class);

lazyMiddleware — клас резолвиться з контейнера під час запиту. Детальніше — Middleware.

Іменовані маршрути

Кожному маршруту можна задати ім’я для генерації URL:

$router->get('/login', [CabinetController::class, 'showLogin'])->setName('cabinet.login');

У PHP (через UrlGeneratorInterface):

$url = $urlGenerator->route('cabinet.login');
$path = $urlGenerator->path('admin.user.edit', ['id' => 5]);

У Twig (розширення UrlExtension):

{{ route('cabinet.login') }}
{{ path('admin.user.edit', { id: user.id }) }}
{{ base_url() }}

Маршрути компонентів

Компонент реалізує ComponentInterface::routes() і повертає абсолютний шлях до файлу. Приклад: src/Components/AuthAdmin/routes.php з префіксами /admin, /admin/users.

Усі маршрути потрапляють в один Router — уникайте дублювання імен (setName) між файлами.

Перегляд маршрутів (CLI)

php bin/console.php route:list
php bin/console.php route:list -F   # повні імена middleware

Виводить метод, URI, ім’я, handler і middleware для кожного зареєстрованого маршруту.

Поради

  • Тримайте публічні маршрути в routes/web.php, модульні — у Component/routes.php.
  • Завжди іменуйте маршрути, які з’являються у шаблонах або редіректах.
  • POST/PUT/DELETE з формами — разом із VerifyCsrfTokenMiddleware і CSRF-полем.
  • Захищені зони оформлюйте group() + auth-middleware, а не перевіркою в кожному контролері.
