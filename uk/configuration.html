Конфігурація

Перед цим розділом перегляньте Налаштування — bootstrap і paths.php. Concept Core збирає налаштування з PHP-файлів у каталозі config/, далі застосовує оверрайди для середовища (APP_ENV) і змінні з .env. Доступ у коді — через ConfigInterface (контейнер DI).

Порядок завантаження

  1. Усі PHP-файли з config/ (бібліотека Noodlehaus Config).
  2. Файл .env у корені проєкту (vlucas/phpdotenv).
  3. Оверрайд з config/{APP_ENV}/, якщо така папка існує (наприклад config/dev/).
  4. Злиття змінних .env у конфіг (найвищий пріоритет).

Після завантаження встановлюється часовий пояс з app.timezone.

Значення APP_ENV зчитується з .env до злиття змінних у конфіг і використовується лише для вибору папки оверрайду config/{APP_ENV}/.

Мапінг .env → конфіг

Кожна змінна оточення перетворюється на ключ конфігу за правилом «перше підкреслення»: ім’я ділиться на дві частини (ліміт explode('_', ..., 2)), переводиться в нижній регістр, далі з’єднується крапкою. 

APP_DEBUG=true     → app.debug
DB_HOST=127.0.0.1   → db.host
LOG_MAX_FILES=14    → log.max_files

Глибше першого рівня через .env не задається: APP_FOO_BAR_BAZ стане app.foo_bar_baz, а не вкладеним масивом. Складні структури — у PHP-файлах config/.

Булеві значення в .env (true, false, 0, 1) коректно читаються через getBool().

Оверрайд за APP_ENV

Якщо існує каталог config/{значення APP_ENV}/, усі PHP-файли з нього зливаються поверх базового config/. Імена файлів і структура масивів — такі самі, змінюються лише потрібні ключі.

У skeleton за замовчуванням:

  • config/dev/ — app.debug = true
  • config/production/ — app.debug = false
config/
├── app.php
├── dev/
│   ├── app.php          # debug: true
│   └── components.php   # список компонентів
└── production/
    ├── app.php          # debug: false
    └── components.php   # порожній список

Доступ у коді

Інжектуйте Concept\Core\Components\Config\Contracts\ConfigInterface:

public function __construct(
    private readonly ConfigInterface $config,
) {}

$debug = $this->config->getBool('app.debug');
$name = $this->config->getString('app.name');
$routes = $this->config->get('routes', []);

Методи: get(), has(), set(), all(), getString(), getInt(), getBool().

Поради для розробки

  • Локально встановіть APP_ENV=dev і APP_DEBUG=true у .env.
  • Секрети (паролі, API-ключі) — лише в .env, не в config/*.php.
  • Спільні значення за замовчуванням — у config/; відмінності середовища — у config/dev/ або config/production/.
  • Нові змінні оточення дотримуйтесь конвенції GROUP_NAME → group.name.
  • Очистіть кеш Twig після зміни шаблонів: php bin/console.php view:clear.
