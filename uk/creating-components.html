Створення компонента

Покроковий приклад на базі AuthAdmin: вхід адміністратора, dashboard, CRUD користувачів, міграції, сидер і CLI-команда.

Передумови: концепція компонентів, глобальний middleware у routes/web.php.

1. Каталог і клас компонента

// src/Components/AuthAdmin/AuthAdminComponent.php
namespace Concept\Components\AuthAdmin;

use Concept\Components\BaseComponent;
use Concept\Core\Services\Component\Contracts\ComponentInterface;

class AuthAdminComponent extends BaseComponent implements ComponentInterface
{
    protected string $componentDir = __DIR__;
}

$componentDir вказує на корінь модуля — звідси читається config.php і будується шлях до routes.php.

2. config.php модуля

// src/Components/AuthAdmin/config.php
return [
    'name' => 'AuthAdmin',
    'description' => 'Admin authentication, dashboard and user management.',
    'version' => '1.0.0',
    'seeders' => [
        'list' => [UserSeeder::class],
    ],
    'migrations' => [
        'paths' => ['src/Components/AuthAdmin/Database/Migrations'],
    ],
    'commands' => [UserListCommand::class],
    'view' => [
        'extensions' => [TwigExtension::class],
        'paths' => [
            'auth-admin' => '/src/Components/AuthAdmin/Views',
        ],
        'contexts' => [
            '/admin' => 'dashboard',
        ],
    ],
    'assets' => [
        'src/Components/AuthAdmin/Assets/admin-tokens.js'
            => 'public/components/auth-admin/js/admin-tokens.js',
    ],
];
  • view.paths — шаблон @auth-admin/login → Views/login.twig
  • view.contexts — URL під /admin отримують route_namespace = dashboard (див. Views)
  • providers — опційно, як у DebugBar

3. Реєстрація в застосунку

// config/components.php
use Concept\Components\AuthAdmin\AuthAdminComponent;

return [
    'components' => [
        AuthAdminComponent::class => AuthAdminComponent::class,
    ],
];

4. Маршрути

// src/Components/AuthAdmin/routes.php
/** @var Router $router */

$router->get('/admin/login', [AdminController::class, 'showLogin'])->setName('admin.login');
$router->post('/admin/login', [AdminController::class, 'login'])->setName('admin.login.submit');

$router->group('/admin', function (RouteGroup $route) {
    $route->get('/dashboard', [AdminController::class, 'index'])->setName('admin.dashboard');
    $route->get('/logout', [AdminController::class, 'logout'])->setName('admin.logout');
})->lazyMiddleware(AuthMiddleware::class);

$router->group('/admin/users', function (RouteGroup $route) {
    $route->get('/', [UserController::class, 'index'])->setName('admin.users');
    $route->post('/store', [UserController::class, 'store'])->setName('admin.user.store');
    // ...
})->lazyMiddleware(AuthMiddleware::class);

Публічний login — без AuthMiddleware; захищені групи — з middleware модуля. Імена маршрутів (admin.*) використовуються в redirectByName() і Twig uri().

5. Middleware авторизації

// Middlewares/AuthMiddleware.php — перевірка сесії, роль admin, статус ACTIVE
public function process(...): ResponseInterface
{
    if (!$this->auth->check()) {
        return new RedirectResponse('/admin/login', HttpStatusCode::FOUND);
    }
    // ...
    return $handler->handle($request);
}

Клас реєструється в DI автоматично; League Route створює екземпляр через контейнер.

6. Контролер і FormRequest

// Controllers/AdminController.php
public function login(LoginRequest $request): ResponseInterface
{
    $loginDto = $request->toDto();
    if ($loginDto && $this->auth->attempt($loginDto->email, $loginDto->password, $loginDto->remember)) {
        return $this->response->redirectByName('admin.dashboard');
    }
    return $this->response->redirectByName('admin.login');
}

public function showLogin(): ResponseInterface
{
    return $this->viewResponse->create('@auth-admin/login');
}

LoginRequest — правила валідації + DTO; CSRF і глобальний HandleValidationExceptionMiddleware працюють як для будь-якого контролера.

Модульні шаблони — @auth-admin/...; спільний chrome адмінки — @dashboard/... з resources/views/dashboard/.

8. База даних

Міграція в Database/Migrations/V2026_03_24_120000_CreateUsersTable.php, модель UserModel, сидер UserSeeder.

php bin/console.php db:migrate
php bin/console.php db:seed -c "Concept\Components\AuthAdmin\Database\Seeders\UserSeeder"

Для одного сідера вкажіть повний FQCN класу в лапках (опція -c або --class). Коротке ім’я на кшталт UserSeeder не працює. Без опції db:seed запускає всі зареєстровані сидери.

Шлях міграцій з config.php потрапляє в MigrationRegistry при boot компонента.

9. Assets і CLI

php bin/console.php component:publish-assets
php bin/console.php component:list

Чеклист нового компонента

  1. {Name}Component.php + config.php
  2. Запис у config/components.php
  3. routes.php (за потреби)
  4. Controllers, Services, Models, Requests/Dto
  5. Views + view.paths / view.contexts
  6. Migrations / seeders / commands / assets — за потреби
  7. db:migrate, component:publish-assets, route:list
